# LingShu v5.2.0 — Terminal Runtime Isolation

**RFC:** [RFC-0052 Terminal Runtime Isolation](../RFC-0052-TERMINAL-RUNTIME-ISOLATION.md)

---

## Overview

v5.2.0 是一次架构级重构，将 LingShu 从"单一服务进程"升级为多运行模式架构。核心变化是分离 **Server Runtime** 和 **Terminal Runtime** 的生命周期。

### 以前的架构

```
main()
 ├── init_tracing()          ← stdout
 ├── init_runtime()          ← federation + config watcher + HTTP
 ├── federation.start()
 ├── config watcher
 ├── ctrl+c handler
 └── if --cil { run_cil() } ← TUI 藏在后台 Runtime 里
```

问题：
- CIL 模式启动完整 Server Runtime（不必要）
- 30+ 行 tracing 日志污染 TUI 终端屏幕
- 后台 tokio 任务与终端竞争 stdout
- 退出时缺少 RAII 保证，Termux 可能残留 raw mode

### 现在的架构

```
main()
 ├── detect_mode()
 ├── Cil
 │   ├── init_cil_logging()   ← logs/cil.log
 │   ├── TerminalGuard::new() ← RAII 终端生命周期
 │   ├── run_tui(token)       ← CancellationToken 优雅退出
 │   └── drop(guard)          ← 自动恢复终端
 └── Server
     ├── init_tracing()       ← stdout (不变)
     ├── init_runtime()
     └── http / repl / headless
```

## 新增功能

### RunMode 路由 (`main.rs`)

- `RunMode::Server` — 现有完整服务模式（默认）
- `RunMode::Cil` — 终端 TUI 模式（跳过 Runtime 初始化）
- `RunMode::Batch` — 批处理模式（预留）
- 新增模式无需继续堆 `if-else` 到 `main.rs`

### TracingProfile (`observability/src/tracing.rs`)

- `TracingProfile::Server` — stdout/stderr（原有行为）
- `TracingProfile::Cil` — 文件 sink：`logs/cil.log`
  - 不含 ANSI 颜色转义
  - 不含 target 前缀（减少噪音）
  - 级别：INFO（默认）
- `TracingProfile::Test` — 简洁输出（预留）

### TerminalGuard RAII (`cli/mod.rs`)

```
TerminalGuard::new()
  ├── enable_raw_mode()
  ├── EnterAlternateScreen
  └── terminal.clear()

TerminalGuard::drop() ← 任何退出路径都会触发
  ├── terminal.clear()
  ├── disable_raw_mode()
  └── LeaveAlternateScreen
```

保证：panic / Ctrl+C / `?` 错误传播 → 终端必然恢复。

### CancellationToken 优雅关闭

- `tokio::sync::CancellationToken` 统一管理 CIL 生命周期
- Ctrl+C → `cancel_token.cancel()` → `run_tui` 主循环退出
- 为后续 Agent Worker、Memory Runtime 提供统一 shutdown 接口

## 兼容性

- **向后兼容**：所有现有 CLI 参数 (`--cil`, `--repl`, `--headless`, `serve`) 行为不变
- **无 crate 边界破坏**：全部变更在 `app`、`observability`、`tests` 三个 crate 内
- **无新增依赖**：`CancellationToken` 使用 tokio 1.38+ 内置实现

## 升级说明

无需手动迁移。`lingshu --cil` 自动使用新路径。
